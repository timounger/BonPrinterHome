Configuration

Note: To configure the BonPrinter tool you must be logged in as Admin.

Take the following steps for the configuration.

1. Set User

To print the bons the user must be logged in. A pin can be assigned to each user. Service staff user are automatically logged out after the printing process.

Assign the name and pin (only numbers allowed) of the user to the fields. The following options are available in the Configuration → User menu bar.

  • Edit: Edit the current user
  • Import user file: Choose an user configuration file to import
  • Export user file: Export the current user
  • Reset: Edit the default user

2. Set Articles

Assign the name and price of the articles to each of the fields. The following options are available from the Configuration → Articles menu bar.

  • Edit: Edit the current articles
  • Import articles file: Choose an articles configuration file to import
  • Export articles file: Export the current articles
  • Reset: Edit the default articles

3. Select Printer

The available COM ports are displayed in the menu bar Configuration → Printer COM-Port. Select the COM port to which the printer is connected. Select None to use the functions without a printer.

4. Select Paper Width

Select the desired paper width to format the printouts correctly.

5. Select SmartCard Reader

If you want to use a smart card reader for user authentication, you can select your reader.

6. Restart und Clear

  • Restart the program to ensure that all components have been correctly initialised.
  • Create a report to clear the list (see help chapter Report).

Optional Settings

Select Output Path

You can change the output path of report files.

Sound

User input generates sounds as feedback. The sound can be enabled or disabled in the menu Settings → Sound.

Theme

Change theme design Settings → Theme.

Log Verbosity

In order to track bugs in the software, important information is recorded in the log file bonprinter_log.log (same path as BonPrinter program). This file records error messages when the program crashes. You can increase the information output here.

  • (0) Error: only fatal errors that cause the program to crash are logged.
  • (1) Warning: all messages highlighted in orange in the status bar are also logged. These are invalid user entries that the software recognises.
  • (2) Info: all messages in the status bar are also logged. These are entries that have been made.
  • (3) Debug: additional debug messages are logged. e.g. updates of views
